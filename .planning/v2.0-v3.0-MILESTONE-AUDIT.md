### Integration Issues Found

**1. Pool Double-Login Trap (Advisory)**
`Pop3ConnectionManager::connect()` calls `builder.connect().await?` then `client.login()` unconditionally. If a caller passes a `Pop3ClientBuilder` with `.credentials()` already set, the builder auto-authenticates during `connect()`, and the subsequent `login()` call fails with `NotAuthenticated`. All documented usage and tests pass plain builders, so this is not a break under normal use — but the API permits misuse without a guard.

**2. fetch_unseen Performance Gap (Advisory)**
`fetch_unseen()` retrieves messages sequentially via individual `retr()` calls, even when the server supports pipelining. The docstring explicitly documents this and directs callers to use `unseen_uids()` + `retr_many()` for pipelined throughput. This is a design decision, not a bug, but means the Phase 5→6 pipelining synergy is caller-managed rather than library-internal.
