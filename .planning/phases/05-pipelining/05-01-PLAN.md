## Context

Phase 5 adds RFC 2449 command pipelining. This first plan prepares the Transport layer: upgrading the writer to BufWriter for batched flushes, adding the `ConnectionClosed` error variant needed by Phase 7, adding the `is_closed` flag for Phase 8 connection pooling, and making reader/writer fields `pub(crate)` for direct access in pipelined batch methods.

### User Decisions (from CONTEXT.md -- NON-NEGOTIABLE)

- **Per-item results**: `retr_many` returns `Vec<Result<Message>>`, `dele_many` returns `Vec<Result<()>>` (Plan 02)
- **I/O errors mid-pipeline**: return successfully-received results so far (Plan 02)
- **`pub fn supports_pipelining(&self) -> bool`** read-only accessor (Plan 02)
- **CAPA probe for PIPELINING happens automatically during `login()`** (Plan 02)
- **Window size constant W = 4** (Claude's discretion, Plan 02)

### Claude's Discretion (from CONTEXT.md)

- **BufWriter buffer size**: Use default (8 KB) -- commands are ~15 bytes each, no bottleneck
- **Error variant wording for ConnectionClosed**: `"connection closed"` -- matches the existing EOF message
- **`is_closed` field**: private `bool` on Transport, set on EOF in `read_line()`

## Tasks

<task id="1" title="Upgrade Transport writer to BufWriter and add is_closed field">

### What
Wrap the Transport `writer` field in `tokio::io::BufWriter`, add an `is_closed: bool` field, update all Transport construction sites (connect_plain, connect_tls variants, mock, upgrade_in_place), and make reader/writer fields `pub(crate)` for direct access by pipelined batch methods.

### Why
BufWriter is required for efficient pipelined command batching -- multiple `write_all` calls accumulate in the buffer before a single `flush()` sends them as one TCP write. The `is_closed` flag is infrastructure for Phase 8 (bb8 connection pool health check). Making fields `pub(crate)` enables batch methods in `client.rs` to directly write commands without going through `send_command()`.

### How

**Step 1: Add `ConnectionClosed` variant to `src/error.rs`**

Add this variant to `Pop3Error`, after the `Timeout` variant:

```rust
/// The server closed the connection unexpectedly.
///
/// This typically indicates the server dropped the connection (e.g., due
/// to inactivity timeout or server shutdown). Downstream phases use this
/// variant for reconnection logic.
#[error("connection closed")]
ConnectionClosed,
```

No change needed -- `Pop3Error` is already re-exported as `pub use error::Pop3Error`, and `ConnectionClosed` is a variant of that enum. Callers can already match on `Pop3Error::ConnectionClosed`.

```

**Step 8: Update `upgrade_in_place`**

The current code does:
```rust
let old_writer = std::mem::replace(&mut self.writer, placeholder_write);
```

After the BufWriter upgrade, the `placeholder_write` needs to also be wrapped in BufWriter. And when recovering the raw `WriteHalf` from `old_writer`, we need `into_inner()`.

Change the placeholder creation from:
```rust
let (placeholder_read, placeholder_write) = io::split(InnerStream::Upgrading);

let old_reader = std::mem::replace(&mut self.reader, BufReader::new(placeholder_read));
let old_writer = std::mem::replace(&mut self.writer, placeholder_write);

let read_half = old_reader.into_inner();
let inner_stream = read_half.unsplit(old_writer);
```

to:

```rust
let (placeholder_read, placeholder_write) = io::split(InnerStream::Upgrading);

let old_reader = std::mem::replace(&mut self.reader, BufReader::new(placeholder_read));
let old_writer = std::mem::replace(&mut self.writer, BufWriter::new(placeholder_write));

let read_half = old_reader.into_inner();
let write_half = old_writer.into_inner();
let inner_stream = read_half.unsplit(write_half);
```

And the reconstruction after TLS handshake, change from:
```rust
let (new_read, new_write) = io::split(tls_inner);
self.reader = BufReader::new(new_read);
self.writer = new_write;
```

to:

```rust
let (new_read, new_write) = io::split(tls_inner);
self.reader = BufReader::new(new_read);
self.writer = BufWriter::new(new_write);
```

<task id="2" title="Add is_closed() public accessor on Pop3Client and update quit()">

### What
Add a `pub fn is_closed(&self) -> bool` method on `Pop3Client` that delegates to `Transport::is_closed()`. Update `quit()` to set `is_closed` on the transport before dropping. Add tests.

### Why
Success criterion 5 requires `is_closed() -> bool` on the client. Phase 8 (bb8 connection pooling) uses this for health checks. `quit()` should mark the connection as closed so any future health check (if a pool somehow retains a reference) sees a closed state.

### How

**Step 1: Add `is_closed()` method to `Pop3Client` in `src/client.rs`**

Add this method after `is_encrypted()`:

```rust
/// Returns `true` if the connection is known to be closed.
///
/// This flag is set when the server closes the connection (EOF) or
/// after [`quit()`](Self::quit) completes. It is not a live probe --
/// a connection that has been silently dropped by the server without
/// sending EOF will still return `false`.
///
/// # Example
///
/// ```no_run
/// use pop3::Pop3Client;
///
/// #[tokio::main]
/// async fn main() -> pop3::Result<()> {
///     let client = Pop3Client::connect_default("pop.example.com:110").await?;
///     assert!(!client.is_closed());
///     client.quit().await?;
///     // client is consumed by quit() -- cannot check is_closed() after
///     Ok(())
/// }
/// ```
pub fn is_closed(&self) -> bool {
    self.transport.is_closed()
}
```

**Step 2: Make `Transport::is_closed` settable for quit()**

Add a `set_closed` method to Transport (in the main `impl Transport` block, not the test block):

```rust
/// Mark the transport as closed (called by quit() after successful QUIT).
pub(crate) fn set_closed(&mut self) {
    self.is_closed = true;
}
```

**Step 3: Update `quit()` in `src/client.rs`**

Add `this.transport.set_closed();` after the successful QUIT response:

```rust
pub async fn quit(self) -> Result<()> {
    let mut this = self;
    this.transport.send_command("QUIT").await?;
    let line = this.transport.read_line().await?;
    response::parse_status_line(&line)?;
    this.transport.set_closed();
    Ok(())
    // `this` is dropped here, TCP connection closes
}
```

**Step 4: Add client tests for is_closed**

Add to the `client.rs` tests module:

```rust
#[tokio::test]
async fn is_closed_false_on_new_client() {
    let mock = Builder::new().build();
    let client = build_test_client(mock);
    assert!(!client.is_closed());
}

#[tokio::test]
async fn is_closed_true_after_eof() {
    let mock = Builder::new().build(); // empty -- EOF on first read
    let mut client = build_authenticated_test_client(mock);
    let result = client.stat().await;
    assert!(result.is_err());
    assert!(client.is_closed());
}
```

### Verification

```bash

cargo test -- quit
cargo clippy -- -D warnings
cargo fmt --check
```

`is_closed()` returns false on a fresh client and true after EOF. `quit()` marks transport as closed. All existing tests pass.
