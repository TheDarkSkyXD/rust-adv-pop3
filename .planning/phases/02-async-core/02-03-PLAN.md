Use `dtolnay/rust-toolchain@stable` (NOT the unmaintained `actions-rs/*`).

Per locked decisions:
- Stable Rust only (MSRV defined in Phase 3)
- Ubuntu runner only (cross-platform in Phase 3)
- Default features only (TLS matrix deferred to Phase 3 — QUAL-04)
- Three jobs: test, clippy, fmt

```yaml
name: CI

on:
  push:
  pull_request:

env:
  CARGO_TERM_COLOR: always

jobs:
  test:
    name: Test
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: dtolnay/rust-toolchain@stable
      - uses: Swatinem/rust-cache@v2
      - run: cargo test

  clippy:
    name: Clippy
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: dtolnay/rust-toolchain@stable
        with:
          components: clippy
      - uses: Swatinem/rust-cache@v2
      - run: cargo clippy -- -D warnings

  fmt:
    name: Format
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: dtolnay/rust-toolchain@stable
        with:
          components: rustfmt
      - run: cargo fmt --check
```

Notes:
- `Swatinem/rust-cache@v2` caches the target directory for faster CI runs (widely used, zero config).
- Three independent jobs (test, clippy, fmt) run in parallel on GitHub Actions.
- The TLS feature flag matrix (QUAL-04) is deferred to Phase 3 when TLS code exists. The workflow structure supports adding `strategy: matrix:` to the test job at that time.
- The `.travis.yml` file in the repo root is for the old v1 codebase. Do NOT delete it in this plan — it's part of the original repo history. Phase 3 may clean it up.
