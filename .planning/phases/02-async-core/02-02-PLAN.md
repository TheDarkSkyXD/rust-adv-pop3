**Step 1: Add SessionState to types.rs**

Add the `SessionState` enum at the top of src/types.rs (before Stat):

```rust
/// The current state of the POP3 session.
#[derive(Debug, Clone, PartialEq, Eq)]
pub enum SessionState {
    /// Connected to the server but not yet authenticated.
    Connected,
    /// Successfully authenticated (TRANSACTION state in RFC 1939).
    Authenticated,
    /// Session has ended (after QUIT).
    Disconnected,
}
```

All existing types already derive Debug (API-03 satisfied). SessionState also derives Debug.

**Step 2: Rewrite src/client.rs to async**

Replace the entire client.rs with an async version. Key changes:

1. **Imports:** Replace `std::net::ToSocketAddrs` with nothing (tokio's ToSocketAddrs is used internally in Transport). Add `use std::time::Duration;`. Keep all crate imports.

2. **Pop3Client struct:**
   ```rust
   pub struct Pop3Client {
       transport: Transport,
       greeting: String,
       state: SessionState,
   }
   ```
   Replace `authenticated: bool` with `state: SessionState`.

3. **connect():** Now `pub async fn`. Signature change — remove TlsMode parameter for Phase 2 (plain TCP only), add timeout parameter:
   ```rust
   /// Connect to a POP3 server over plain TCP.
   ///
   /// The `timeout` duration is applied to every read operation for the lifetime
   /// of this connection. Use `DEFAULT_TIMEOUT` for a sensible default (30s).
   pub async fn connect(
       addr: impl tokio::net::ToSocketAddrs,
       timeout: Duration,
   ) -> Result<Self>
   ```
   - Calls `Transport::connect_plain(addr, timeout).await?`
   - Reads greeting via `transport.read_line().await?`
   - Sets `state: SessionState::Connected`
   - Returns `Pop3Client`

   Also provide a convenience constructor:
   ```rust
   /// Connect with the default timeout (30 seconds).
   pub async fn connect_default(addr: impl tokio::net::ToSocketAddrs) -> Result<Self> {
       Self::connect(addr, crate::transport::DEFAULT_TIMEOUT).await
   }
   ```

   **Note on TlsMode:** Remove the TlsMode enum from client.rs for Phase 2. Phase 3 will reintroduce TLS connection methods. This is consistent with CONTEXT.md: "Phase 2 is plain TCP only — TLS is Phase 3." Remove `pub enum TlsMode` and the match on it in connect(). Update lib.rs to remove the TlsMode re-export.

4. **state() accessor:**

   ```rust
       self.state.clone()

   ```

5. **greeting() accessor:** Unchanged (not async).

6. **login():** Now `pub async fn`. Check state first:
   ```rust
   pub async fn login(&mut self, username: &str, password: &str) -> Result<()> {
       if self.state != SessionState::Connected {
           return Err(Pop3Error::NotAuthenticated);
           // Could use a more specific error, but NotAuthenticated covers
           // "you can't login when already authenticated or disconnected"
       }
       check_no_crlf(username)?;
       check_no_crlf(password)?;
       self.send_and_check(&format!("USER {username}")).await
           .map_err(|e| match e {
               Pop3Error::ServerError(msg) => Pop3Error::AuthFailed(msg),
               other => other,
           })?;
       self.send_and_check(&format!("PASS {password}")).await
           .map_err(|e| match e {
               Pop3Error::ServerError(msg) => Pop3Error::AuthFailed(msg),
               other => other,
           })?;
       self.state = SessionState::Authenticated;
       Ok(())
   }
   ```

7. **All authenticated methods** (stat, list, uidl, retr, dele, rset, noop, top, capa): Add `async` keyword, add `.await` to every `self.send_and_check()` and `self.transport.read_multiline()` call. Change `require_auth()` to check `self.state == SessionState::Authenticated`.

8. **quit(self):** Takes ownership per locked decision:
   ```rust
   /// End the session. Messages marked for deletion are removed.
   ///
   /// This method consumes the client, preventing any further use.
   /// If the caller drops the client without calling quit(), the TCP
   /// connection closes silently and pending DELE marks are NOT committed.
   pub async fn quit(self) -> Result<()> {
       let mut this = self;
       this.transport.send_command("QUIT").await?;
       let line = this.transport.read_line().await?;
       response::parse_status_line(&line)?;
       // this.state = SessionState::Disconnected; // Optional: client is dropped after this
       Ok(())
       // `this` is dropped here, TCP connection closes
   }
   ```
   Note: `quit()` no longer uses `send_and_check` (which takes `&mut self`) — it inlines the logic since `self` is consumed.

9. **send_and_check():** Now `async fn`:
   ```rust
   async fn send_and_check(&mut self, cmd: &str) -> Result<String> {
       self.transport.send_command(cmd).await?;
       let line = self.transport.read_line().await?;
       let text = response::parse_status_line(&line)?;
       Ok(text.to_string())
   }
   ```

10. **require_auth():** Update to use SessionState:
    ```rust
    fn require_auth(&self) -> Result<()> {
        if self.state != SessionState::Authenticated {
            Err(Pop3Error::NotAuthenticated)
        } else {
            Ok(())
        }
    }
    ```

11. **check_no_crlf() and validate_message_id():** Unchanged (pure functions).

**Multi-line response tests** (list_all, uidl_all, retr, top, capa): The mock must script reads as separate `.read()` calls. For multi-line responses, the server sends the status line then the body lines then the dot terminator. Since `read_multiline()` calls `read_line()` in a loop, each call to `read_line()` reads until `\r\n`. With `tokio_test::io::Builder`, you can provide all the data in a single `.read()` call — the BufReader will buffer it and return line by line.

Example for list_all:
```rust

        .write(b"LIST\r\n")

        .build();

    let entries = client.list(None).await.unwrap();
```
