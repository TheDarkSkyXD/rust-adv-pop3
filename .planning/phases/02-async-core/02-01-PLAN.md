<task type="auto">
  <name>Task 2: Rewrite transport.rs to async with tokio I/O and mock via tokio_test</name>
  <files>src/transport.rs</files>
  <action>
Completely rewrite src/transport.rs to use async tokio I/O. The new transport uses `Box&lt;dyn AsyncRead + Unpin + Send&gt;` and `Box&lt;dyn AsyncWrite + Unpin + Send&gt;` for the reader/writer (not an enum), enabling both real TCP and test mock to use the same code path.

**Structure:**

```rust
use std::time::Duration;
use tokio::io::{self, AsyncBufReadExt, AsyncWriteExt, BufReader};
use tokio::net::TcpStream;
use crate::error::{Pop3Error, Result};

/// Default read timeout for POP3 operations (30 seconds).
pub(crate) const DEFAULT_TIMEOUT: Duration = Duration::from_secs(30);

pub(crate) struct Transport {
    reader: BufReader<Box<dyn io::AsyncRead + Unpin + Send>>,
    writer: Box<dyn io::AsyncWrite + Unpin + Send>,
    timeout: Duration,
}
```

**Methods (all async):**

1. `pub(crate) async fn connect_plain(addr: impl tokio::net::ToSocketAddrs, timeout: Duration) -> Result&lt;Self&gt;`:
   - Use `tokio::net::TcpStream::connect(addr).await?`
   - Use `tokio::io::split(stream)` to get `(ReadHalf, WriteHalf)`
   - Wrap ReadHalf in `BufReader::new(Box::new(read_half))`
   - Store WriteHalf as `Box::new(write_half)`
   - Store timeout duration

2. `pub(crate) async fn connect_tls(addr: impl tokio::net::ToSocketAddrs, hostname: &str, timeout: Duration) -> Result&lt;Self&gt;`:
   - Keep the existing TLS logic but adapt for async. Since rustls `StreamOwned` does not implement tokio's AsyncRead/AsyncWrite natively, and Phase 3 will properly handle TLS with tokio-rustls, for Phase 2 this method should return `Pop3Error::Io` with a message "TLS not yet supported in async mode — use Phase 3". This is acceptable because Phase 2 is plain TCP only per the CONTEXT.md boundary ("TLS backends ... are separate phases").
   - Alternative (preferred): Keep the method signature for API compatibility but immediately error. This prevents compile errors in client.rs TlsMode match.

3. `pub(crate) async fn send_command(&mut self, cmd: &str) -> Result&lt;()&gt;`:
   - `self.writer.write_all(cmd.as_bytes()).await?`
   - `self.writer.write_all(b"\r\n").await?`
   - `self.writer.flush().await?`

4. `pub(crate) async fn read_line(&mut self) -> Result&lt;String&gt;`:
   - Wrap the actual read in `tokio::time::timeout(self.timeout, ...)`:
   ```rust
   let mut line = String::new();
   let n = tokio::time::timeout(self.timeout, self.reader.read_line(&mut line))
       .await
       .map_err(|_| Pop3Error::Timeout)?  // Elapsed -> Timeout
       .map_err(Pop3Error::Io)?;          // io::Error -> Io (via From)
   // Actually: .map_err(|e| Pop3Error::Io(e))? or just ? since From<io::Error> exists
   ```
   Wait — `BufReader::read_line` returns `io::Result<usize>`. The `?` with `From<io::Error>` should work directly:
   ```rust
   let n = tokio::time::timeout(self.timeout, self.reader.read_line(&mut line))
       .await
       .map_err(|_| Pop3Error::Timeout)??;
   ```
   The double `?` unwraps: outer `Result<Result<usize, io::Error>, Elapsed>` -> map_err converts Elapsed -> `Result<Result<usize, io::Error>, Pop3Error>` -> first `?` unwraps Pop3Error -> `Result<usize, io::Error>` -> second `?` converts io::Error via From.

   If n == 0, return `Pop3Error::Io(io::Error::new(UnexpectedEof, "connection closed"))`.

5. `pub(crate) async fn read_multiline(&mut self) -> Result&lt;String&gt;`:
   - Same dot-unstuffing logic as current sync version, but calls `self.read_line().await?` in the loop.

**Mock constructor (test-only):**

```rust
#[cfg(test)]
impl Transport {
    pub(crate) fn mock(mock: tokio_test::io::Mock) -> Self {
        let (r, w) = tokio::io::split(mock);
        Transport {
            reader: BufReader::new(Box::new(r)),
            writer: Box::new(w),
            timeout: Duration::from_secs(30),
        }
    }
}
```

Note the change: the old mock returned `(Transport, Rc&lt;RefCell&lt;Vec&lt;u8&gt;&gt;&gt;)` for write inspection. With `tokio_test::io::Builder`, write expectations are baked into the mock itself — the builder `.write(expected_bytes)` panics if the actual write differs. So there's no need to return a separate writer handle. Tests will now rely on the mock's built-in write validation instead of manual byte inspection.

**Transport tests:**

Replace the existing `test_dot_unstuffing_logic` test with an async version:

```rust
#[cfg(test)]
mod tests {
    use super::*;
    use tokio_test::io::Builder;

    #[tokio::test]
    async fn test_dot_unstuffing_via_transport() {
        let mock = Builder::new()
            .read(b"..This had a leading dot\r\n")
            .read(b"Normal line\r\n")
            .read(b".\r\n")
            .build();
        let mut transport = Transport::mock(mock);
        let body = transport.read_multiline().await.unwrap();
        assert!(body.contains(".This had a leading dot"));
        assert!(!body.contains("..This"));
        assert!(body.contains("Normal line"));
    }

    #[tokio::test]
    async fn test_send_command() {
        let mock = Builder::new()
            .write(b"STAT\r\n")
            .build();
        let mut transport = Transport::mock(mock);
        transport.send_command("STAT").await.unwrap();
        // tokio_test mock validates the write automatically
    }

    #[tokio::test]
    async fn test_read_line_eof() {
        let mock = Builder::new().build(); // empty — EOF
        let mut transport = Transport::mock(mock);
        let result = transport.read_line().await;
        assert!(result.is_err());
    }
}
```

**IMPORTANT:** Remove all `use std::io::{BufRead, BufReader, Write}` and `use std::net::{TcpStream, ToSocketAddrs}` from transport.rs. Remove `use std::cell::RefCell`, `use std::io::Cursor`, `use std::rc::Rc`. Remove the `set_timeouts` helper function entirely (tokio does not support socket-level timeouts). Remove the `Stream` enum entirely. Remove `use std::sync::Arc` (the Arc for rustls config is no longer needed in transport.rs since TLS is stubbed for Phase 2).

Keep the `use rustls::...` imports ONLY if the `connect_tls` method still references rustls types. If `connect_tls` is stubbed to return an error, remove the rustls imports from transport.rs (they'll come back in Phase 3).

**Decision (Claude's Discretion):** Use a single `Duration` timeout field stored in Transport, applied to every `read_line()` call. Named `timeout`. Passed via the `connect_plain(addr, timeout)` parameter. The `DEFAULT_TIMEOUT` constant is public(crate) so client.rs can use it as a default.
