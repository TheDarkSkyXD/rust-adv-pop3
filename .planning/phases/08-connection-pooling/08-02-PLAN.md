**Key decisions from CONTEXT.md (locked):**
- Pool is a registry of per-account bb8 pools, each with `max_size(1)` -- NOT a traditional N-connection pool
- `tokio::sync::RwLock<HashMap>` for the account-to-pool registry (no DashMap -- avoids new dependency, uses already-available tokio)
- Inner `Arc<bb8::Pool<M>>` mandatory -- clone Arc while briefly holding the map reference, then release the map guard before any `.await`
- `build_unchecked()` for synchronous pool creation inside the lock -- avoids holding the RwLock across an `.await` during pool build
- `min_idle(None)` -- connections created lazily on first checkout
- bb8's `PooledConnection` exposed directly (type alias), not wrapped in a newtype
- Accounts can be added dynamically at any time via `pool.add_account()`
- Busy checkout blocks with configurable timeout (bb8's `connection_timeout`)
- Pool documentation must prominently quote RFC 1939 section 8

**From RESEARCH.md (critical patterns):**
- Inner Arc pattern: `Arc<bb8::Pool<M>>` inside the HashMap. Clone the Arc while briefly holding the read lock, then release the read lock, then call `.get().await` on the cloned Arc. NEVER hold the RwLock guard across `.await`.
- `retry_connection(false)` -- auth failures propagate immediately instead of looping until timeout
- `build_unchecked()` creates pool synchronously without initial connection -- critical for avoiding async inside write lock
- DashMap vs RwLock: use RwLock per CONTEXT.md to avoid adding another dependency

**From RESEARCH.md (anti-patterns to avoid):**
- NEVER hold RwLock guard across `.await` points
- NEVER set `max_size > 1` -- RFC 1939 violation
- NEVER call `.build().await` while holding the write lock -- use `build_unchecked()` instead

**From CONTEXT.md (Claude's discretion, resolved here):**
- Implement `remove_account()` for deprovisioning -- allows callers to clean up accounts
- Idle timeout default: 5 minutes (300s) -- POP3 servers typically timeout idle connections at 10 minutes; 5 min allows proactive recycling
- Max lifetime default: 30 minutes -- consistent with bb8's own default
- Connection timeout default: 30 seconds -- consistent with bb8's own default
- Expose `bb8::PooledConnection` via type alias `PooledConnection<'a>` -- ergonomic, documents the Deref<Target = Pop3Client> behavior
- Failed connections on return: bb8 auto-discards broken connections via has_broken() -- no explicit handling needed

  ```rust
  /// A connection pool for managing multiple POP3 mailbox accounts concurrently.
  ///
  /// # RFC 1939 Exclusive Mailbox Access
  ///
  /// **POP3 forbids concurrent access to the same mailbox.** Per RFC 1939 section 8:
  ///
  /// > "the POP3 server then acquires an exclusive-access lock on the maildrop,
  /// > necessary to prevent messages from being overwritten by stranded
  /// > retrievals, and stranded removes (stranding stranding stranding)."
  ///
  /// > "If the maildrop cannot be opened for some reason (for example, a lock
  /// > can not be acquired, the user is denied access to the maildrop, or the
  /// > maildrop cannot be parsed), the POP3 server responds with a negative
  /// > status indicator."
  ///
  /// This pool enforces that constraint at the library level: each mailbox
  /// account is backed by an independent pool capped at **one connection**.
  /// A caller attempting to check out a connection to an account that is already
  /// in use will **wait asynchronously** until the previous caller drops their
  /// [`PooledConnection`].
  ///
  /// This is a **per-account** model, not a traditional N-connection pool.
  /// Multiple accounts can be accessed concurrently; a single account cannot.
  ///
  /// # Usage
  ///
  /// ```no_run
  /// use pop3::pool::{Pop3Pool, AccountKey};
  /// use pop3::Pop3ClientBuilder;
  ///
  /// #[tokio::main]
  /// async fn main() -> Result<(), Box<dyn std::error::Error>> {
  ///     let pool = Pop3Pool::new();
  ///
  ///     // Register an account
  ///     let key = AccountKey::new("pop.example.com", 110, "alice");
  ///     pool.add_account(
  ///         key.clone(),
  ///         Pop3ClientBuilder::new("pop.example.com").port(110),
  ///         "alice",
  ///         "secret",
  ///     );
  ///
  ///     // Check out a connection (blocks if already in use by another task)
  ///     let mut conn = pool.checkout(&key).await?;
  ///     let stat = conn.stat().await?;
  ///     println!("{} messages", stat.message_count);
  ///     // Connection returns to pool when `conn` drops
  ///
  ///     Ok(())
  /// }
  /// ```
  pub struct Pop3Pool {
      pools: RwLock<HashMap<AccountKey, Arc<bb8::Pool<Pop3ConnectionManager>>>>,
      config: PoolConfig,
  }
  ```

  IMPORTANT DESIGN CORRECTION: The registry map should use `std::sync::RwLock`, NOT `tokio::sync::RwLock`. Reasons:
  1. `add_account()` should be a synchronous function (no await needed -- `build_unchecked` is sync)
  2. The lock is held only for brief HashMap operations (insert, get, remove, keys) -- never across await points

  4. `checkout()` acquires a read lock, clones the `Arc<Pool>`, releases the lock immediately, THEN awaits -- the lock is NEVER held across await

  impl Pop3Pool {
      pub fn new() -> Self {
          Self {
              pools: RwLock::new(HashMap::new()),
              config: PoolConfig::default(),
          }
      }

      pub fn with_config(config: PoolConfig) -> Self {
          Self {
              pools: RwLock::new(HashMap::new()),
              config,
          }
      }

      pub fn add_account(
          &self,
          key: AccountKey,
          builder: Pop3ClientBuilder,
          username: impl Into<String>,
          password: impl Into<String>,
      ) {
          let mut pools = self.pools.write().expect("pool registry lock poisoned");
          if pools.contains_key(&key) {
              return; // idempotent
          }
          let manager = Pop3ConnectionManager::new(builder, username, password);
          let pool = bb8::Pool::builder()
              .max_size(1)
              .min_idle(Some(0))
              .test_on_check_out(true)
              .retry_connection(false)
              .connection_timeout(self.config.connection_timeout)
              .idle_timeout(self.config.idle_timeout)
              .max_lifetime(self.config.max_lifetime)
              .build_unchecked(manager);
          pools.insert(key, Arc::new(pool));
      }

      pub async fn checkout(&self, key: &AccountKey) -> Result<PooledConnection, Pop3PoolError> {
          let pool = {
              let pools = self.pools.read().expect("pool registry lock poisoned");
              pools
                  .get(key)
                  .cloned() // clones the Arc, not the pool
                  .ok_or_else(|| Pop3PoolError::UnknownAccount(key.clone()))?
          };
          // RwLock guard dropped here -- safe to await
          pool.get_owned().await.map_err(Pop3PoolError::from)
      }

      pub fn remove_account(&self, key: &AccountKey) -> bool {
          let mut pools = self.pools.write().expect("pool registry lock poisoned");
          pools.remove(key).is_some()
      }

      pub fn accounts(&self) -> Vec<AccountKey> {
          let pools = self.pools.read().expect("pool registry lock poisoned");
          pools.keys().cloned().collect()
      }
  }
