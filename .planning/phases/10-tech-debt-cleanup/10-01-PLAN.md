# Plan 10-01: Tech Debt Cleanup

## Goal

Close all advisory gaps from the v2.0+v3.0 milestone audit: remove stale `#[allow(dead_code)]` annotations and plan-phase references from shipped source, and add a double-login guard to the pool connection manager so `login()` is not called redundantly when the builder already auto-authenticated.

## must_haves

These derive from the phase success criteria and must ALL be true after execution:

1. `Pop3ConnectionManager::connect()` checks `client.state() != SessionState::Authenticated` before calling `login()` — preventing redundant USER/PASS when the builder auto-authenticates
2. The three stale `#[allow(dead_code)]` annotations on `upgrade_in_place`, `tls_handshake` (rustls), and `tls_handshake` (openssl) are removed from `src/transport.rs`
3. The two surviving `#[allow(dead_code)]` annotations (`Upgrading` variant line 28, `connect_tls` stub line 228) remain unchanged
4. Zero occurrences of "Plan 0" or "Plan 1" exist in any `src/*.rs` file
5. `Pop3ConnectionManager` rustdoc describes the auth guard behavior
6. A new unit test in `src/pool.rs` verifies the authenticated-client precondition the guard relies on
7. `cargo clippy --features rustls-tls,pool,mime -- -D warnings` passes
8. `cargo test --features rustls-tls,pool,mime` passes
9. `cargo build --no-default-features --features pool` passes (no-TLS feature build)

<title>Add double-login guard and auth rustdoc to Pop3ConnectionManager</title>
<active_form>Adding double-login guard to pool connection manager</active_form>
<details>

**1. Add `SessionState` import.**

Add `use crate::types::SessionState;` to the imports at the top of the file (after the existing `use crate::error::Pop3Error;` line).

**2. Update `Pop3ConnectionManager` rustdoc to describe the auth guard.**

BEFORE (line 59-63):
```rust
/// Manages POP3 client connections for a single account via bb8.
///
/// Each manager stores a cloned [`Pop3ClientBuilder`] and credentials.
/// On [`connect()`](bb8::ManageConnection::connect), it clones the builder,
/// connects, and authenticates via USER/PASS.
```

AFTER:
```rust
/// Manages POP3 client connections for a single account via bb8.
///
/// Each manager stores a cloned [`Pop3ClientBuilder`] and credentials.
/// On [`connect()`](bb8::ManageConnection::connect), it clones the builder,
/// connects, and authenticates via USER/PASS.
///
/// # Authentication
///
/// If the builder already authenticated the client (because credentials were
/// configured on the builder via
/// [`Pop3ClientBuilder::credentials()`](crate::Pop3ClientBuilder::credentials)),
/// the `login()` step is skipped automatically. The client's
/// [`SessionState`](crate::SessionState) is checked after `connect()` to
/// prevent issuing a redundant `USER`/`PASS` exchange.
```

**3. Add the double-login guard to `connect()` method.**

The CRLF check MUST remain first (defense-in-depth). The state check is inserted between `builder.connect()` and `client.login()`.

BEFORE (inside the async move block, after the CRLF check):
```rust
            let mut client = builder.connect().await?;
            client.login(&username, &password).await?;
            Ok(client)
```

AFTER:
```rust
            let mut client = builder.connect().await?;
            // Guard: builder.connect() auto-authenticates when credentials are
            // configured on the builder. Skip login() if the session is already
            // in Authenticated state to prevent a redundant USER/PASS exchange.
            if client.state() != SessionState::Authenticated {
                client.login(&username, &password).await?;
            }
            Ok(client)
```

**4. Add a unit test verifying the authenticated-client precondition.**

Add the following test inside the `#[cfg(test)] mod tests` block, after the existing `has_broken_returns_true_for_closed_client` test:

```rust
    // --- double-login guard tests ---

    #[tokio::test]
    async fn authenticated_client_state_is_authenticated() {
        // Validates the precondition the double-login guard relies on:
        // build_authenticated_mock_client produces SessionState::Authenticated.
        let mock = tokio_test::io::Builder::new().build();
        let client = build_authenticated_mock_client(mock);
        assert_eq!(client.state(), SessionState::Authenticated);
    }
```

Also add `use crate::types::SessionState;` inside the test module (after the existing `use super::*;` and `use crate::client::build_authenticated_mock_client;` lines).
